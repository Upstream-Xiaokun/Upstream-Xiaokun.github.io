部署 Hexo 博客到 GitHb 的方法

发表于2021-11-17|更新于2021-11-17
|字数总计:598|阅读时长:2分钟|阅读量:

确保你的 git 已经安装好

1、右键打开 Git Brash Here

2、在 Git Brash Here 输入

git config --global user.name yourname

git config --global user.email  youremail

3、检查是否输入正确

git config user.name

git config user.email

4、创建 SSH 密钥

ssh-keygen -t rsa -C youremail

5、打开.ssh的文件夹Owner

路径：C:\Users\minec.ssh

不同的电脑位置可能有所不同，自行查询

6、打开 id_rsa.pub 公共密钥

注意 id_rsa 为私人密钥，不能给别人

7、在Github 上配置密钥

Setting 为 Github 个人设置

SSH and GPG keys 配置密钥

再将刚刚打开的 id_rsa.pub 公共密钥里面的内容复制，点击 New SSH key

这里第一栏的 Title 名字随意，第二栏粘贴刚刚复制的内容，最后点击 Add SSH key 添加成功

如果SSH密钥添加成功，你的邮箱将收到一条来自 GitHub 邮件

7、检测电脑端是否部署成功

在 Git Brash 中输入，看到有 successfully 的单词就说明部署成功·

8、创建属于自己的 GitHub 仓库

点开右上角的头像，选择 Your repositories

点击 New 创建新的仓库

Owner 代表的是你的 GitHub 名字，Repository name 代表的是仓库名字

（注意：仓库名字格式必须为 你的 GItHub 名字.github.io）不然网页可能打不开

9、修改部署配置文件

打开 myblog 文件夹里面的站点配置文件 _config.yml，翻到最后

修改里面的内容为

repo 中第一个 YourgithubName 填写的是你的 GitHub 名字（Owner），第二个 YourgithubName.github.io 填的是你仓库（Repository name）

branch：属于仓库分支，默认就行，不用改动

10、安装deploy-git

安装deploy-git后，也就是部署的命令,这样你才能用命令部署到GitHub。
npm install hexo-deployer-git --save

报错请更换下载镜像源（自行百度）

11、再次打开 myblog 文件夹，右键 Git Brash Here 打开

按照顺序依次执行clean 初始化 myblog 里面的文件，清除之前生成的东西；hexo g 生成静态文件，hexo d 部署文件到 GitHb 的仓库里面

hexo clean
hexo g
hexo d

依次执行完，如果没有报错，就部署文件到 GitHub 仓库里面了

12、打开网站

http://yourgithubname.github.io（yourgithubname 你的用户名或者仓库名，二者名字一致）
